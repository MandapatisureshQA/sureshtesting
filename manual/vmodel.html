V - Model

Advantages of V Model

Testing is involved in each and every phase.

Disadvantages of V Model

  1. Documentation is more.
  2. Initial investment is more.

Verification V/S Validation

Verification checks whether we are building the right product

  1. Focus on Documentation
  2. Verification typically involves
    1. Reviews
    2. Walkthroughs
    3. Inspections

2. Verification Techniques or Static Testing Techniques

   2.1. Reviews

     Conducts on documents to ensure Correctness and Completeness

  1. Requirement Reviews
  2. Design Reviews
  3. Code Reviews
  4. Test Plan Reviews
  5. Test Cases Reviews etc..

    2.2. Walkthrough

  1. It is informal review
  2. Author reads the documents or code and discuss with peers.
  3. It is not preplanner and can be done whenever required.
  4. Also, walkthrough does not have minutes of the meet.

    2.3. Inspection

  1. It’s a most formal review type
  2. In which at least 3-8 people will sit in the meeting 1-reader, 2-writer, 3-moderator plus concerned.
  3. Inspection will have a proper schedule which will be intimated via E-Mail to the concerned developer/ Tester.

Validation checks whether we are building the product right

  1. Takes place after verifications are completed.
  2. Focus on Software
  3. Validation typically involves actual testing - Unit Testing, Integration, System Testing, User Acceptance Testing
